Recitation 5

Developing your classes:

Below is the main() for this scenario. Your classes must provide the functionality as shown in the test code.
Consider doing incremental development:

  1. comment out all the code in main() except the seven output statements
  2. start writing the class definition for LabWorker - write only the data members and the constructor and get this compiling
  3. to test that LabWorker really works, what would you uncomment in main()? {the two LabWorker variable defintions}
    You would use the debugger to check the values stored in a lab worker's data member are correct - or you might want to write a display method - for today - don't write the display method in LabWorker - just use the debugger to check that the correct name was set and that NULL is where it should be (where should it be?)
  4. start writing the class definition for Section - write only the data members and the constructor and get this compiling
  5. to test Section, what would you uncomment in main()? {the two Section variable definitions}
  6. what else needed to be written?
  7. write the class TimeSlot and try to compile again
  8. add the display() method to Section (and any other methods needed elsewhere)
  9. to test what you just added, what you would you uncomment in main()? {secA2.display() and secB3.display() }
    NOTE: the output should be NO RECORDS since no records have been added and NONE for the lab worker since no lab worker has been associated yet (you would NOT set "NONE" as the name of the lab worker when there is not one connected).
    This is the same output as after resetting the Section shown at the bottom of the output below
  10. write the LoadStudentsFromFile() method in the Section class
  11. test this by uncommenting both calls to this method in main()
    NOTE: now the output should be more like what is shown in the output below before any grades are added - but the lab worker should still say NONE
  12. now try to get the association between LabWorker and Section working - write the addInstructor() method in Section (and write any other methods needed in other places)
  13. to test what you just added, uncomment both addInstructor() calls
    NOTE: now the output should completely match the output below for the two sections' displays before any grades are added
  14. now write the displayGrades() method in LabWorker
  15. to test what you just added, uncomment both calls to this method
    NOTE: the output of the grades should be NO RECORDS since no records have been added.
    This is the different than after resetting the Section because the lab worker is still associated with the section:
    Joe has section: A2; Tuesday 4PM
    NO RECORDS
  16. now write the AddGrade() method (and any other methods needed)
  17. to test, uncomment all of the code for modeling week one and week two
  18. now write the reset() method in Section (and any other methods needed)
  19. to test, uncomment the last lines of code at the bottom of main() - all of main() should be uncommented now

You are not required to go through these steps but if you do, we believe you will have a good chance of learning things, perhaps better than if you just hacked away in any old order of developement.
